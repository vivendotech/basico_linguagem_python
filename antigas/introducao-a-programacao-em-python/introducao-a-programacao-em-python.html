Podemos definir que um paradigma de programação é uma forma de classificar as linguagens de programação conforme os seus recursos. Começaremos discutindo o paradigma de programação estruturada. Em seguida começaremos a discutir o paradigma da orientação ao objeto . Para continuar vale relembrar o que é uma linguagem de programação – conjunto de palavras reservadas e regras de como usar essas palavras ; e qual era o sistema de tipagem do Python – quais eram as regras para atribuir um tipo a uma variável. .

O paradigma da programação estruturada é um paradigma de programação que tem dois recursos fundamentais. Ele tem recursos que permitem a criação e a utilização de blocos de códigos, e recursos que permitem controlar o fluxo de uso dos blocos de código do programa. Nesse post iremos discutir os blocos de códigos em Python. E no próximo iremos começar a discutir os controladores de fluxo (e.g. if, for).

introdução a programação em Python: Blocos de códigos em Python

No Python os blocos de códigos são determinados a partir da identação do código escrito. No python um novo bloco de código começa quando há uma nova identação do código (4 espaços). E um bloco de códigos termina na primeira linha de código escrito numa identação anterior (retorno de 4 espaços).[4] E todas as linhas de códigos que estão no mesmo nível de identação pertecem ao mesmo bloco de código. Algumas outras linguagens (e.g c, c++ , java) fazem a determinação de seus blocos de códigos usando os parenteses {} . Inicialmente, nosso novo script tudo é um único bloco de código. Um exemplo didático pode ser visto na figura 1.

Figura 1 – O primeiro bloco de código do Python é o próprio documento!

Nos próximo tutorial iremos discutir sobre as outras palavras reservadas do Python que também geram blocos de códigos (e.g. if, for, while). Não discutimos elas nesse tutorial porque essas palavras reservadas também tem a função de controlar o fluxo da execução do código. Então, para separar apenas o tema de blocos de código nesse tutorial iremos discuti-los utilizando apenas os blocos de códigos produzidos pelas funções.

Estruturação dos blocos código

No tutorial sobre uma linguagem de programação discutimos a palavra reservada “def” e como usa-la para cria uma função. Na verdade quando você está criando uma função em Python o que você está fazendo é determinando um bloco de código e reservando uma palavra para o chamar/invocar. No Python em um único arquivo você pode ter vários blocos de códigos definidos.

Vamos então criar nosso primeiro bloco de código e o invocá-lo. Segue o código 1. 


        # código 1

        # bloco de código 1

        def primeiraFuncao():

            # bloco de código 1.1

            pass

        # bloco de código 1

        primeiraFuncao()

O código 1 tem dois blocos de códigos. Sendo que um deles está dentro do outro. O primeiro bloco de código é o próprio documento e o segundo é a função “primeiraFuncao”. A estrutura desses dois blocos está apresentada de forma mais didática na figura 2 :

Figura 2 – Quando criamos uma função, também criamos um novo bloco de códigos. A estrutura que o documento fica é essa!

Blocos de códigos aninhados

Como você pode ver na figura 2 os blocos de códigos podem ser estruturados de formas aninhadas. Ou seja, os blocos de programação podem ter vários blocos dentro deles

Para entender melhor essa afirmação vamos refatorar o código 1 e adicionar a ele uma segunda função dentro da primeira função . O código 1 refatorado fica igual ao código 2. 


        # Código 2

        # bloco de código 1

        # 1

        def primeiraFuncao():

           # bloco de código 1.1

           def primeiraFuncaoDentroDeOutra():

              # bloco de código 1.1.1

              print("helloMundo")

           # bloco de código 1.1

           primeiraFuncaoDentroDeOutra()

Ao analisar a estrutura do nosso código percebemos que o bloco principal (chamado de um) tem um bloco definido dentro dele. Esse bloco de código chamamos de “primeiraFuncao”. Dentro do bloco 1.1 intitulado “primeirafuncao” definimos um bloco de códigos chamado de “primeiraFuncaoDentroDeOutra”. Um exemplo didático dessa afirmação pode ser visto na figura 3:

Figura 3 – Análise de estrutura de blocos do código 2

Agora dentro da identação do código 1 iremos invocar a “primeiraFuncao()”. Para isso iremos adicionar o código conforme o código 3. 


        # Código 3 - Invocando uma função

        # bloco de código 1

        primeiraFuncao()

        # helloMundo

O resultado escrito em nosso prompt sera “helloMundo”. Agora para testar iremos tentar chamar a função primeiraFuncaoDentroDeOutra dentro do primeiro bloco de código. O código criado será como o descrito no código 3. 


        # Código 3 -Invocando uma função

        # bloco de código 2

        primeiraFuncaoDentroDeOutra()

        # primeiraFuncaoDentroDeOutra' is not defined

O resultado da execução do código 3 será não existe essa função definida. Mas, o código 2 funcionou porque será que esse código não funcionou? Esse código não funcionou porque a definição da função “primeiraFuncaoDentroDeOutra” só vive dentro do bloco que código no qual ela foi denifinida.

As funções só vivem dentro de seu bloco de código

Dentro dos blocos de códigos você pode definir tanto novas funções quanto novas variáveis. Isso independente do nível que esses blocos de códigos forem definidos. Porém, é super importante perceber que blocos de códigos com menor identação (mais próximos da identação 0) não tem acesso as variáveis e funções dos blocos de códigos com maior identação. Isso acontece porque as variáveis e as funções só ficam definidas dentro do bloco de programação que elas foram criadas. Esse tópico fica mais claro no post sobre variáveis locais e globais.
